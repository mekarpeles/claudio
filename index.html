Works with cmux

The session manager

claudio is the messaging layer. cmux is the session manager — it spawns named Claude Code agents in persistent tmux windows and wires their inboxes together.

Use both to run a whole team of named agents that coordinate on their own.

→ github.com/mekarpeles/cmux
# spin up a named agent, detached cmux up alice -d -- "You are Alice, a code reviewer" # message it from anywhere cmux send alice "please review PR #42"
# each agent gets its own issue tracker cq issue create -t "review PR #42" cq issue list cq issue close 1
cq — task queue

Give your agents a to-do list.

They'll actually use it. cq gives every agent a personal issue tracker that lives in its home dir and follows it across restarts.

Auto-resolves to the current agent inside a cmux session — no setup, no flags.

Experiments

Primitives, not frameworks

Small, composable pieces. Wire them together however your topology demands.

claudio nudge

Self-reminders at idle

Register a prompt that fires as a reminder when the agent goes quiet. "Have you checked your cq today? Anything new for your identity.md?"

claudio pair

Synchronous discovery

Two agents exchange socket addresses in a blocking handshake. No registry, no broker — just a direct peer-to-peer link.

claudio start

Live inbox session

Drop into a live inbox right from your terminal. Anything sent to you appears in real time — great for ad-hoc agent-to-human messaging.

claudio send

Fire-and-forget delivery

Write a message to any agent's socket from a script, a cron job, or another agent. It's queued and delivered the moment they're idle.
